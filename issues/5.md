# Database Normalization Project

This document outlines the comprehensive details regarding the Database Normalization Project.

## 1. Project Overview
The Database Normalization Project aims to structure the database to reduce data redundancy and improve data integrity.

## 2. Phases of Database Normalization
1. **Unnormalized Form (UNF)**  
   - Collect raw data without any structure.
2. **First Normal Form (1NF)**  
   - Ensure that each column has atomic values and each entry is unique.
   - Example: Splitting full names into first and last names.
3. **Second Normal Form (2NF)**  
   - Remove partial dependencies.  
   - Ensure all non-key attributes are fully functional dependent on the primary key.
4. **Third Normal Form (3NF)**  
   - Remove transitive dependencies.  
   - Non-key attributes should not depend on other non-key attributes.
5. **Boyce-Codd Normal Form (BCNF)**  
   - Ensure that every dependency is a functional dependency.
6. **Fourth Normal Form (4NF)**  
   - Eliminate multi-valued dependencies.
7. **Fifth Normal Form (5NF)**  
   - Ensure that data is stored in a way that any non-trivial join dependency is implied by the candidate keys.
